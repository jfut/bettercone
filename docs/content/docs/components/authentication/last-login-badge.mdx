# LastLoginBadge

The `LastLoginBadge` component displays a small badge indicating which authentication method was last used by the user. Perfect for improving UX by highlighting familiar sign-in options.

<Callout type="info">
  Requires the Better Auth [Last Login Method plugin](https://www.better-auth.com/docs/plugins/last-login-method) to be enabled on your server.
</Callout>

## Installation

```tsx
import { LastLoginBadge } from '@bettercone/ui';
```

## Features

- ✅ **Automatic Detection** - Reads from Better Auth's lastLoginMethod plugin
- ✅ **Provider Icons** - Shows icons for OAuth providers (Google, GitHub, etc.)
- ✅ **Customizable** - Custom text, variants, and styling
- ✅ **Smart Display** - Only renders when a method is available
- ✅ **Type-Safe** - Full TypeScript support

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `method` | `string` | - | Authentication method to display (e.g., "google", "email", "github"). If not provided, automatically reads from authClient |
| `className` | `string` | - | Additional CSS classes |
| `variant` | `"default" \| "secondary" \| "outline" \| "destructive"` | `"secondary"` | Badge style variant |
| `showIcon` | `boolean` | `true` | Show provider icon alongside text |
| `text` | `string` | `"Last used"` | Custom badge text |
| `localization` | `Partial<AuthLocalization>` | - | Custom localization strings |

## Supported Authentication Methods

The component automatically displays the appropriate name and icon for these methods:

- **OAuth Providers**: `google`, `github`, `discord`, `facebook`, `apple`, etc.
- **Email**: `email`
- **Passkey**: `passkey`
- **Magic Link**: `magic-link`
- **Phone**: `phone`
- **Anonymous**: `anonymous`
- **SIWE**: `siwe` (Sign-In with Ethereum)
- **Custom**: Any custom authentication method (displays capitalized)

## Examples

### Automatic Method Detection

```tsx
import { LastLoginBadge } from '@bettercone/ui';
import { AuthUIProvider } from '@bettercone/ui';

export function SignInPage() {
  return (
    <AuthUIProvider authClient={authClient}>
      <div className="space-y-4">
        <h1>Sign In</h1>
        
        {/* Badge automatically reads from authClient */}
        <LastLoginBadge />
      </div>
    </AuthUIProvider>
  );
}
```

### With AuthView Integration

```tsx
"use client";

        </div>

      <AuthView authClient={authClient} />
    </div>
  );
}
```

## Setup

To use this component, you need to enable the Last Login Method plugin in your Better Auth configuration:

### Server Setup

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { lastLoginMethod } from "better-auth/plugins"

export const auth = betterAuth({
    plugins: [
        lastLoginMethod({
            // Optional: Store in database for persistence
            storeInDatabase: true,
            
            // Optional: Custom cookie settings
            cookieName: "better-auth.last_used_login_method",
            maxAge: 60 * 60 * 24 * 30, // 30 days
        })
    ]
})
```

### Client Setup

```ts title="auth-client.ts"
import { createAuthClient } from "better-auth/client"
import { lastLoginMethodClient } from "better-auth/client/plugins"

export const authClient = createAuthClient({
    plugins: [
        lastLoginMethodClient({
            cookieName: "better-auth.last_used_login_method"
        })
    ]
})
```

## Best Practices

1. **Highlight Familiar Methods**: Use the badge to make previously used sign-in methods more prominent
2. **Combine with Button Variants**: Change button appearance based on last method used
3. **Progressive Enhancement**: Badge only shows when method is available - no errors if plugin missing
4. **Consistent Placement**: Place badges consistently (usually top-right or right-aligned)

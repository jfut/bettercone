# StorageUsageCard

The `StorageUsageCard` component displays real-time storage usage with human-readable formatting (GB/MB/KB), visual progress indicators, and upgrade prompts when approaching capacity limits.

<Callout type="info">
  This component automatically converts bytes to the most appropriate unit (B, KB, MB, GB, TB) for display.
</Callout>

## Installation

```tsx
import { StorageUsageCard } from '@bettercone/ui';
```

## Features

- ✅ **Real-Time Tracking** - Live updates via Convex subscriptions
- ✅ **Smart Formatting** - Automatic byte conversion (B, KB, MB, GB, TB)
- ✅ **Visual Progress** - Color-coded progress bars
- ✅ **Warning System** - Alerts at configurable thresholds
- ✅ **Upgrade Prompts** - Contextual upgrade suggestions
- ✅ **Percentage Display** - Clear usage percentage
- ✅ **Skeleton Loading** - Smooth loading states
- ✅ **Type-Safe** - Full TypeScript support

## Byte Formatting

The component automatically formats bytes to human-readable units:

```typescript
const formatBytes = (bytes: number): string => {
  if (bytes === 0) return "0 B";
  
  const k = 1024;
  const sizes = ["B", "KB", "MB", "GB", "TB"];
  const i = Math.floor(Math.log(bytes) / Math.log(k));
  
  return `${(bytes / Math.pow(k, i)).toFixed(2)} ${sizes[i]}`;
};

// Examples:
// 1024 → "1.00 KB"
// 1048576 → "1.00 MB"
// 1073741824 → "1.00 GB"
// 750000000 → "715.26 MB"
```

### Example 4: With Upload Progress

```tsx
import { StorageUsageCard } from '@/components/usage/storage-usage-card';
import { Progress } from '@/components/ui/progress';
import { useState } from 'react';

export default function StoragePage() {
  const [uploadProgress, setUploadProgress] = useState(0);
  const [currentStorage, setCurrentStorage] = useState(8000000000);

  const handleUpload = async (file: File) => {
    // Simulate upload
    setUploadProgress(0);
    
    for (let i = 0; i <= 100; i += 10) {
      await new Promise(resolve => setTimeout(resolve, 200));
      setUploadProgress(i);
    }
    
    // Update storage
    setCurrentStorage(prev => prev + file.size);
  };

  return (
    <div className="space-y-6">
      <StorageUsageCard
        currentBytes={currentStorage}
        limitBytes={10737418240}
        onUpgrade={() => router.push('/pricing')}
      />

        <Card>
          <CardHeader>
            <CardTitle>Uploading...</CardTitle>
          </CardHeader>
          <CardContent>
            <Progress value={uploadProgress} />
          </CardContent>
        </Card>
      )}
    </div>
  );
}
```

## Storage Tracking

Track file uploads in your application:

```typescript
// In your file upload handler
import { api } from '@/convex/_generated/api';
import { fetchMutation } from 'convex/nextjs';

export async function POST(request: Request) {
  const formData = await request.formData();
  const file = formData.get('file') as File;
  
  // Check storage limit before upload
  const usage = await fetchQuery(api.usage.getCurrentUsage, {
    userId: session.user.id,
  });
  
  if (usage.storageBytes + file.size > usage.storageLimit) {
    return Response.json(
      { error: 'Storage limit exceeded' },
      { status: 413 } // Payload Too Large
    );
  }
  
  // Upload file...
  const fileUrl = await uploadToStorage(file);
  
  // Track storage usage
  await fetchMutation(api.usage.trackStorage, {
    userId: session.user.id,
    bytes: file.size,
    operation: 'add',
  });
  
  return Response.json({ url: fileUrl });
}
```

<Callout type="warning">
  Always validate storage limits server-side before accepting file uploads to prevent abuse.
</Callout>

## Delete Files to Free Space

Implement file deletion with storage tracking:

```typescript
// Delete file handler
export async function DELETE(request: Request) {
  const { fileId } = await request.json();
  
  // Get file size before deletion
  const file = await getFile(fileId);
  
  // Delete file
  await deleteFromStorage(fileId);
  
  // Update storage tracking
  await fetchMutation(api.usage.trackStorage, {
    userId: session.user.id,
    bytes: file.size,
    operation: 'remove', // Decrease storage
  });
  
  return Response.json({ success: true });
}
```

## Notes

- Storage data updates in real-time via Convex
- Byte formatting automatically selects appropriate unit
- Progress bar color changes based on usage percentage
- Warning appears at 80%, critical alert at 90%
- Upgrade button only shows when `onUpgrade` is provided
- All calculations use binary units (1024 bytes = 1 KB)

# OrganizationView

The `OrganizationView` component provides a complete organization management interface with tabbed navigation for settings, members, and API keys.

<Callout type="info">
  This component follows the same pattern as `AccountView` and provides a full-page organization management experience with responsive navigation.
</Callout>

## Installation

```tsx
import { OrganizationView } from '@bettercone/ui';
```

## Features

- ✅ **Tab-Based Navigation** - Settings, Members, and API Keys tabs
- ✅ **Responsive Design** - Desktop sidebar, mobile drawer
- ✅ **Organization Settings** - Logo, name, slug, and deletion
- ✅ **Member Management** - View and manage organization members
- ✅ **Invitation System** - Send and manage invitations
- ✅ **API Key Management** - Create and manage API keys
- ✅ **Deep Linking** - Direct navigation to specific sections
- ✅ **Customizable** - Hide navigation, custom styling
- ✅ **Type-Safe** - Full TypeScript support

## Three-Tier Pattern

OrganizationView follows the **Three-Tier Architecture** pattern:

### Tier 1: Self-Contained (Context)

Uses `AuthUIProvider` context to automatically fetch organization data:

```tsx
import { AuthUIProvider } from '@bettercone/ui';
import { OrganizationView } from '@bettercone/ui';

export default function App() {
  return (
    <AuthUIProvider authClient={authClient}>
      <OrganizationView slug="my-org" />
    </AuthUIProvider>
  );
}
```

### Tier 2: Agnostic (Custom Client)

Provide a custom auth client for different backends:

```tsx
import { OrganizationView } from '@bettercone/ui';
import { createCustomAuthClient } from './custom-auth';

export default function OrganizationPage() {
  const customClient = createCustomAuthClient();
  
  return (
    <OrganizationView 
      slug="my-org"
      // Component will use context from AuthUIProvider
    />
  );
}
```

### Tier 3: Manual Props (Presentational)

The organization slug is required for fetching data:

```tsx
import { OrganizationView } from '@bettercone/ui';

export default function OrganizationPage() {
  return (
    <OrganizationView 
      slug="my-organization"
      view="settings"
    />
  );
}
```

## Navigation Structure

The component provides three main sections:

### Settings Tab

Contains organization settings cards:
- **Logo Card** - Upload and manage organization logo
- **Name Card** - Edit organization display name
- **Slug Card** - Change organization URL slug
- **Delete Card** - Delete organization with confirmation

### Members Tab

Contains member management:
- **Members Card** - View and manage organization members
- **Invitations Card** - Send and manage pending invitations

### API Keys Tab

Contains API key management:
- **API Keys Card** - Create, view, and revoke API keys

### Next.js with Dynamic Views

```tsx title="app/organizations/[slug]/[[...view]]/page.tsx"
import { OrganizationView } from '@bettercone/ui';

export default function OrganizationPage({ 
  params 
}: { 
  params: { slug: string; view?: string[] } 
}) {
  const view = params.view?.[0] as OrganizationViewPath | undefined;
  
  return (
    <OrganizationView 
      slug={params.slug}
      view={view}
    />
  );
}
```

## Permissions

The component respects Better Auth organization permissions:

- **Owner** - Full access to all tabs and actions
- **Admin** - Can manage members and API keys, limited settings access
- **Member** - Read-only access, cannot modify settings

Buttons and actions are automatically disabled based on user role.

## Localization

All strings are customizable via the `localization` prop:

```tsx
<OrganizationView 
  slug="my-org"
  localization={{
    SETTINGS: "Settings",
    MEMBERS: "Members", 
    API_KEYS: "API Keys",
    // ... more strings
  }}
/>
```

See the [AuthLocalization](/docs/guides/localization) guide for all available strings.

## Composition

OrganizationView is composed of these components:

- `OrganizationSettingsCards` - Logo, name, slug, delete cards
- `OrganizationMembersCard` - Member list and management
- `OrganizationInvitationsCard` - Invitation management
- `ApiKeysCard` - API key management

You can use these individually for custom layouts.

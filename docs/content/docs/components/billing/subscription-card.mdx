# SubscriptionCard

The `SubscriptionCard` component displays the user's current subscription plan with management actions including access to the Stripe Customer Portal.

<Callout type="info">
  This component automatically detects and displays subscriptions for both personal accounts and organizations using Better Auth's Stripe plugin.
</Callout>

## Installation

```tsx
import { SubscriptionCard } from '@bettercone/ui';
```

## Features

- ✅ **Backend-Agnostic** - Works with any Better Auth backend (Convex, Prisma, Supabase, Drizzle)
- ✅ **Auto-Detection** - Automatically finds active subscriptions for user or organization
- ✅ **Stripe Portal** - One-click access to Stripe Customer Portal via callbacks
- ✅ **Status Badges** - Visual indicators for subscription status (active, trialing, past_due)
- ✅ **Action Callbacks** - Customizable actions (manage, cancel, upgrade)
- ✅ **Localization** - Full i18n support with custom labels
- ✅ **Skeleton Loading** - Smooth loading states
- ✅ **Type-Safe** - Full TypeScript support with Better Auth types

## Subscription Detection

The component automatically detects subscriptions by:

1. **Personal Subscriptions**: Matches `subscription.referenceId` to `user.id`
2. **Organization Subscriptions**: Matches `subscription.referenceId` to any organization the user belongs to
3. **Active Status**: Filters for `active` or `trialing` subscriptions

```typescript
// Automatic detection logic
const validReferenceIds = [
  session?.user?.id,          // Personal subscription
  ...organizations.map(org => org.id) // Org subscriptions
];

const subscription = subscriptions.find(
  sub => validReferenceIds.includes(sub.referenceId) &&
         (sub.status === "active" || sub.status === "trialing")
);
```

## Subscription Status

The component displays different UI based on subscription status:

| Status | Badge Color | Description |
|--------|-------------|-------------|
| `active` | Green | Subscription is active and paid |
| `trialing` | Blue | In free trial period |
| `past_due` | Yellow | Payment failed, retry in progress |
| `canceled` | Red | Subscription canceled but still active until period end |
| `incomplete` | Orange | Subscription requires payment action |

## Stripe Customer Portal

The "Manage Billing" button opens Stripe's Customer Portal where users can:

- Update payment methods
- Download invoices
- View billing history
- Cancel subscription
- Update billing information

```tsx
// Stripe portal integration
const handleManageBilling = async () => {
  setIsPortalLoading(true);
  
  const result = await authClient.subscription.createCustomerPortal({
    returnUrl: window.location.href,
  });
  
  if (result.data?.url) {
    window.location.href = result.data.url;
  }
};
```

<Callout type="warning">
  The Stripe Customer Portal must be enabled in your Stripe Dashboard settings before this feature works.
</Callout>

## Error Handling

The component gracefully handles various states:

- **No subscription**: Shows "Get Started" prompt
- **Loading error**: Falls back to empty state with action
- **Portal error**: Shows toast notification

```tsx
// Error handling with toast notifications
authClient.subscription.createCustomerPortal({ returnUrl })
  .then(result => {
    if (result.error) {
      toast.error("Failed to open billing portal");
    }
  });
```

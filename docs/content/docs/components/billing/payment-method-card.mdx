# PaymentMethodCard

The `PaymentMethodCard` component displays the default payment method (credit card) attached to the user's subscription with quick access to update via Stripe Customer Portal.

<Callout type="info">
  This component is backend-agnostic and works with any Better Auth backend. It uses callbacks to handle payment method management.
</Callout>

## Installation

```tsx
import { PaymentMethodCard } from '@bettercone/ui';
```

## Features

- ✅ **Backend-Agnostic** - Works with any Better Auth backend (Convex, Prisma, Supabase, Drizzle)
- ✅ **Secure Display** - Shows last 4 digits and card brand only
- ✅ **Auto-Detection** - Automatically finds payment method from active subscription
- ✅ **Stripe Portal** - One-click update via callback to your Stripe portal
- ✅ **Card Icons** - Visual card brand indicators (Visa, Mastercard, Amex, etc.)
- ✅ **Expiry Display** - Shows card expiration date
- ✅ **No Card State** - Helpful prompt when no payment method exists
- ✅ **Localization** - Full i18n support
- ✅ **Skeleton Loading** - Smooth loading states

## Stripe Portal Integration

Clicking "Update Payment Method" opens the Stripe Customer Portal focused on payment methods:

```tsx
const handleUpdatePaymentMethod = async () => {
  setIsLoading(true);
  
  const result = await authClient.subscription.createCustomerPortal({
    returnUrl: window.location.href,
  });
  
  if (result.data?.url) {
    window.location.href = result.data.url;
  } else {
    toast.error("Failed to open payment portal");
  }
  
  setIsLoading(false);
};
```

<Callout type="warning">
  Payment method updates must be done through Stripe's secure portal. Never attempt to collect card details directly in your application.
</Callout>

### Example 2: With Expiry Warning

```tsx
import { PaymentMethodCard } from '@/components/billing/cards/payment-method-card';
import { Alert, AlertDescription, AlertTitle } from '@/components/ui/alert';
import { AlertTriangle } from 'lucide-react';
import { authClient } from '@/lib/auth-client';

export default function BillingPage() {
  const { data: subscriptions } = authClient.useSession();
  
  // Check if card expires soon (within 30 days)
  const isExpiringSoon = subscriptions?.[0]?.paymentMethod && (
    new Date(
      subscriptions[0].paymentMethod.expYear,
      subscriptions[0].paymentMethod.expMonth - 1
    ) < new Date(Date.now() + 30 * 24 * 60 * 60 * 1000)
  );

  return (
    <div className="space-y-6">
      {isExpiringSoon && (
        <Alert variant="destructive">
          <AlertTriangle className="h-4 w-4" />
          <AlertTitle>Card Expiring Soon</AlertTitle>
          <AlertDescription>
            Your payment method expires soon. Update it to avoid service interruption.
          </AlertDescription>
        </Alert>
      )}
      
      <PaymentMethodCard />
    </div>
  );
}
```

### Example 4: With Manual Update Handler

```tsx
import { PaymentMethodCard } from '@/components/billing/cards/payment-method-card';
import { useState } from 'react';
import { toast } from 'sonner';

export default function BillingPage() {
  const [isUpdating, setIsUpdating] = useState(false);

  const handleUpdate = async () => {
    setIsUpdating(true);
    
    // Custom analytics or logging
    console.log('User initiated payment method update');
    
    try {
      const result = await authClient.subscription.createCustomerPortal({
        returnUrl: `${window.location.href}?updated=true`,
      });
      
      if (result.data?.url) {
        window.location.href = result.data.url;
      }
    } catch (error) {
      toast.error('Failed to open payment portal');
      setIsUpdating(false);
    }
  };

  return (
    <PaymentMethodCard 
      showActions={true}
      className={isUpdating ? 'opacity-50 pointer-events-none' : ''}
    />
  );
}
```

## Security Considerations

<Callout type="warning">
  **PCI Compliance**: Never store or display full card numbers. Always use Stripe's secure portal for payment updates.
</Callout>

- Only last 4 digits are displayed
- Card details come from Stripe's secure API
- All updates go through Stripe Customer Portal
- No sensitive data is stored in your database
- Follows PCI DSS compliance requirements

## Payment Method Types

While credit cards are most common, Stripe supports various payment methods:

| Type | Display |
|------|---------|
| Credit Card | Card brand icon + last 4 + expiry |
| Debit Card | Card brand icon + last 4 + expiry |
| Bank Account | Bank name + last 4 account digits |
| PayPal | PayPal email |
| Apple Pay | "Apple Pay" + last 4 |
| Google Pay | "Google Pay" + last 4 |

<Callout type="info">
  The component currently focuses on card display. For other payment methods, customize the display logic in the component.
</Callout>

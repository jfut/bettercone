# Fixing Better Auth + Convex + Stripe Integration

## Issue Summary

When integrating Better Auth with Convex and Stripe, you encountered several errors:

1. тЬЕ **FIXED**: `ArgumentValidationError` - subscription model not in validator
2. тЬЕ **FIXED**: Index warnings on member table
3. ЁЯФД **IN PROGRESS**: TypeError about undefined 'fields' 
4. ЁЯФД **IN PROGRESS**: 500 Internal Server Errors

## Root Cause

The Better Auth CLI **does not automatically generate composite indexes** when creating schemas. This is a known limitation documented in the [Better Auth Performance Guide](https://www.better-auth.com/docs/guides/optimizing-for-performance).

## Solution Applied

### 1. Added Composite Index for Member Table

The Convex adapter queries `member` table by `[organizationId, userId]` but the CLI only generates individual indexes.

**Added to `schema.ts`:**
```typescript
member: defineTable({
  // ... fields
})
  .index("organizationId", ["organizationId"])
  .index("userId", ["userId"])
  .index("role", ["role"])
  .index("organizationId_userId", ["organizationId", "userId"]), // тЖР Added
```

### 2. Added Indexes for Subscription Table

Better Auth Stripe plugin requires these indexes for efficient queries:

**Added to `schema.ts`:**
```typescript
subscription: defineTable({
  // ... fields
})
  .index("referenceId", ["referenceId"])       // тЖР Added
  .index("status", ["status"])                  // тЖР Added
  .index("stripeCustomerId", ["stripeCustomerId"]) // тЖР Added
  .index("stripeSubscriptionId", ["stripeSubscriptionId"]), // тЖР Added
```

### 3. Documentation Created

Created two files to document this limitation:
- `INDEXES.md` - Detailed guide on required indexes
- Updated `README.md` - Schema generation instructions

## Understanding the Subscription Table

The auto-generated subscription table is **correct and complete** for Convex:

### Why it looks minimal:

1. **`referenceId` serves dual purpose**: Holds either userId OR organizationId
2. **Convex auto-generates `_id`**: No manual `id` field needed
3. **Numbers for timestamps**: Convex uses Unix timestamps, not Date objects
4. **Optional fields**: Allows for various subscription states

### Comparison with documentation:

| Documentation | Generated Schema | Explanation |
|---------------|------------------|-------------|
| `id` | тЭМ (uses `_id`) | Convex auto-generates |
| `userId` | тЭМ (uses `referenceId`) | Unified reference field |
| `organizationId` | тЭМ (uses `referenceId`) | Unified reference field |
| `createdAt` | тЭМ | Convex auto-generates `_creationTime` |
| `updatedAt` | тЭМ | Can be added if needed |
| `periodStart` (Date) | тЬЕ (number) | Unix timestamp |
| Other fields | тЬЕ All present | Correct Convex types |

## Important Workflow Notes

### When Regenerating Schema:

```bash
# 1. Run Better Auth CLI
npx @better-auth/cli generate --config packages/convex/convex/betterAuth/auth.ts --output packages/convex/convex/betterAuth/schema.ts -y

# 2. IMMEDIATELY re-add the indexes manually
#    See INDEXES.md for the exact code to add

# 3. Restart dev server
```

### Why Manual Editing is Acceptable:

While `schema.ts` says "Do not edit this file manually", adding **performance indexes** is an exception because:

1. The Better Auth team acknowledges this limitation
2. They plan to add index generation in the future
3. It's documented in their performance guide
4. Indexes don't affect the data model, only query performance

## Troubleshooting

### If 500 Errors Persist:

1. **Check Convex Logs**: The browser console only shows HTTP status. Check terminal running `pnpm dev` for actual JavaScript errors

2. **Verify Environment Variables**: 
   ```bash
   # In .env.local
   STRIPE_SECRET_KEY=sk_test_...
   NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_...
   SITE_URL=http://localhost:3000
   ```

3. **Check Convex Dashboard**: Look for schema validation errors or index build failures

4. **Test Without Stripe**: Temporarily disable Stripe plugin in `auth.ts` to isolate the issue

5. **Check Network Tab**: Look at the request/response bodies in browser DevTools тЖТ Network

# Better Auth Convex Schema

This directory contains the auto-generated Better Auth schema for Convex, with one important manual addition required for Stripe functionality.

## Schema Generation

To regenerate the base schema, run:

```bash
cd packages/convex
npx @better-auth/cli generate --config convex/betterAuth/auth.ts --output convex/betterAuth/schema.ts -y
```

Or from the project root with environment variables:

```bash
STRIPE_SECRET_KEY="$STRIPE_SECRET_KEY" SITE_URL="$SITE_URL" npx @better-auth/cli generate --config packages/convex/convex/betterAuth/auth.ts --output packages/convex/convex/betterAuth/schema.ts -y
```

## Important: Subscription Table

**The subscription table is NOT automatically generated by the Better Auth CLI when using the Convex adapter.**

This is a known limitation when using Better Auth Stripe plugin with Convex. After running schema generation, you **MUST** manually add the subscription table to `schema.ts`.

### Why Manual Addition is Required

The Better Auth CLI with Convex adapter doesn't detect runtime-conditional plugins (like the Stripe plugin that's added based on environment variables). Therefore, the subscription table must be manually added and maintained.

### Subscription Table Schema

Add this to the `tables` object in `schema.ts`:

```typescript
subscription: defineTable({
  plan: v.string(),
  referenceId: v.string(),
  stripeCustomerId: v.optional(v.union(v.null(), v.string())),
  stripeSubscriptionId: v.optional(v.union(v.null(), v.string())),
  userId: v.string(),
  organizationId: v.optional(v.union(v.null(), v.string())),
  status: v.string(),
  periodStart: v.number(),
  periodEnd: v.number(),
  cancelAtPeriodEnd: v.optional(v.union(v.null(), v.boolean())),
  seats: v.optional(v.union(v.null(), v.number())),
  trialStart: v.optional(v.union(v.null(), v.number())),
  trialEnd: v.optional(v.union(v.null(), v.number())),
  createdAt: v.number(),
  updatedAt: v.number(),
})
  .index("userId", ["userId"])
  .index("organizationId", ["organizationId"])
  .index("referenceId", ["referenceId"])
  .index("stripeCustomerId", ["stripeCustomerId"])
  .index("stripeSubscriptionId", ["stripeSubscriptionId"]),
```

## After Schema Changes

After modifying the schema:

1. The Convex dev server should automatically detect changes and push the schema
2. If it doesn't, restart the Convex dev server
3. Verify in the Convex dashboard that the subscription table exists
